{# Browser-style panel zoom (issue #116). ONE implementation, {% include %}'d into the of every standalone panel page — same shared-partial pattern _command_palette.html uses for the voice/model picker, but placed in (not at the end of ) so the anti-flash apply below runs before the page paints. Why CSS `zoom` and not `transform: scale`: both the packaged window (WebView2/Chromium) and a source-run's browser (Chrome/Edge) are Chromium, and `zoom` reflows layout exactly like native browser zoom — no scrollbar/hit-testing gymnastics `transform` would need. Persistence is localStorage (per-machine, no server round-trip) — a pure view preference, not a `config.toml`/settings_schema setting (see issue #116 "Decisions"). #}
